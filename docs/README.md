# Zetmel AI Knowledge Pack

A structured source-of-truth package for building the portfolio and branding website of **Safraz Razik** and **Zetmel** with an AI coding or content-generation agent.

## Recommended usage

1. Give the AI agent `00-MASTER-CONTEXT.md` first.
2. Add `01-AI-SITE-INSTRUCTIONS.md` as the agent's project instruction or context file.
3. Let the agent read the remaining files when implementing individual pages or sections.
4. Treat `90-VERIFY-BEFORE-PUBLISHING.md` as a blocking checklist.
5. Do not publish claims marked `NEEDS_CONFIRMATION` until Safraz verifies them.

## Confidence labels

- `CONFIRMED_PUBLIC`: Visible on Safraz's current public website, resume or GitHub.
- `CONFIRMED_USER`: Directly stated by Safraz in conversations.
- `INFERRED_POSITIONING`: A safe strategic interpretation, not a historical fact.
- `NEEDS_CONFIRMATION`: Plausible or previously discussed, but must be verified before publishing.
- `DYNAMIC`: A value such as GitHub stars that changes over time.

## Core rule

Position Safraz as an experienced software engineer who uses AI to accelerate professional software delivery—not as a newly trained AI/ML researcher and not as someone who delegates engineering judgment to AI.
