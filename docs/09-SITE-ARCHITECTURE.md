# Recommended Site Architecture

## Primary navigation

- Home
- Work
- Services
- About
- Open Source
- Writing
- Contact

Optional:

- Lab
- Resume

## Home page

### 1. Hero

Communicate:

- Who Safraz is
- What Zetmel does
- Why AI is relevant
- What the visitor should do next

### 2. Proof strip

Include 4–5 credible proof points.

### 3. Selected work

Feature:

1. Motorola commerce modernization
2. Vue File Agent
3. Adsfly product suite
4. Current AI product/recovery work when ready

### 4. Services

Use 4 concise categories:

- Product Development
- Product Recovery
- Modernization
- Technical Review

### 5. AI-assisted process

Show the Understand → Structure → Accelerate → Review → Recover → Ship sequence.

### 6. Open source

Feature Vue File Agent and Rockplate.

### 7. Experience timeline

Keep the homepage version compact. Link to a full career page or resume.

### 8. Testimonials

Use the real testimonials already displayed on Zetmel, with first names only unless permission supports more identity.

### 9. Contact CTA

Offer a simple project/recruiter route.

## Work page

Content filters:

- Enterprise
- Products
- Open Source
- Tools
- AI Lab
- Legacy Modernization

Each item should have:

- Short summary
- Safraz's role
- Technologies
- Challenge
- Contribution
- Result
- Confidence/source metadata in content files, not necessarily shown to visitors

## Services page

Suggested sections:

1. Who engagements are for
2. Service pillars
3. Example engagement shapes
4. Process
5. What Safraz needs from the client
6. FAQ
7. CTA

Do not publish fixed pricing unless Safraz decides on it.

## About page

Suggested order:

1. Current positioning
2. Career story
3. Founder journey
4. Engineering philosophy
5. AI journey
6. Compact timeline
7. Personal interests
8. Resume/contact links

## Open Source page

Feature:

- Vue File Agent
- Rockplate

## Writing/blog page

Recommended content pillars:

### AI-Accelerated Engineering

- What AI-generated applications get wrong after the MVP
- How to structure milestones for coding agents
- When to switch from planning to implementation
- Recovery sprints for unfinished AI-built software
- Senior code review in an AI-first workflow

### Software Architecture

- Structuring a Next.js/NestJS/Prisma application
- Product architecture decisions
- Codebase audits
- Legacy modernization

### Local LLM Engineering

- Running coding models on an RTX 2070 Super
- Ollama/OpenCode workflows
- Model speed vs quality
- Context and token discipline

### Product Engineering

- Why admin panels make a product feel finished or unfinished
- POS workflow completeness
- Designing returns, printing and reports
- UI polish after functional MVP

### Career and Open Source

- Lessons from 15+ years of engineering
- Building and maintaining Vue File Agent
- From internal tools to a template language
- Founder lessons from local software products

## Contact page

Fields:

- Name
- Email
- Company
- What are you building or improving?
- Current stage
- Preferred engagement: project / audit / consulting / role
- Optional budget range
- Optional timeline

Also support direct links to LinkedIn and GitHub.

## Dual-audience handling

The website serves clients and employers. Do not create two disconnected brands.

Use:

- Main CTA: Discuss a product
- Secondary CTA: View resume / Explore experience
- Contact-form engagement type to distinguish enquiries
- Recruiter summary on About/Resume pages
