**Vue.js · TypeScript · 17KB · Zero dependencies · 700+ GitHub stars**

## Overview

Vue File Agent is a file upload component for Vue.js that I designed and built from scratch and released as open source. It was created because every file upload library I evaluated had gaps — poor previews, weak validation, rigid theming or no support for the file types that real applications need to handle.

The component grew into a full-featured library with over 700 GitHub stars, 88 forks, 15 published releases and contributions from the community. It includes two official themes, a plugin-based architecture, an event-driven state machine, drag-and-drop with folder support, resumable uploads via the tus.io protocol, server-side validation handling and official upload-server examples for PHP and Node.js.

A second generation — File Agent v2 — was started as a framework-agnostic rewrite supporting React, Vue and vanilla JavaScript, but development was set aside before completion. The v1 library remains the stable, published version.

## The problem it solved

File upload is one of those UI problems that seems simple until you have to build it for production. Every application needs it, but doing it well requires handling:

- Drag-and-drop with folder support across browsers
- File type and size validation with clear error messages
- Image, video and audio previews that load efficiently
- Upload progress feedback
- Server-side validation errors returned and displayed inline
- File metadata editing (names, descriptions)
- Sortable file lists
- Resumable uploads for large files
- Cross-framework compatibility
- Accessible keyboard interactions

Most existing Vue upload components handled a subset of these well. I wanted one that handled all of them without requiring additional dependencies.

## Technical architecture

Vue File Agent was designed around a separation between file-management logic and UI presentation.

**Core layer.** The core managed file selection, validation, state transitions and upload orchestration. Files moved through a defined lifecycle — selected, validating, queued, uploading, processing, done, error — with explicit state transitions that could be observed and intercepted via events.

**Validation engine.** Configurable rules for file type (`accept`), file size (`maxSize`), and file count (`maxFiles`) were evaluated at selection time. Error messages were customisable per rule and surfaced through the component's event system, allowing applications to display them inline or in their own UI.

**Upload service.** The built-in uploader handled multipart uploads with progress tracking, supporting custom headers, server-side error responses and the tus.io resumable upload protocol. An external uploader could replace the default by handling the file queue directly.

**Presentation layer.** Two official themes — grid view and list view — rendered file records with appropriate previews, progress indicators and action controls. The Vue slot system exposed every visual section for customisation without forking the component.

**File type handling.** Images rendered as thumbnail previews. Videos and audio files displayed native player controls. Other file types showed contextual icons derived from their extension. This distinction — treating each file type according to its nature — was the central design principle behind the component's tagline: "Every file deserves to be treated equally."

## Developer experience

The component was designed to be usable with minimal configuration:

```vue
<template>
  <VueFileAgent :uploadUrl="uploadUrl" v-model="fileRecords"></VueFileAgent>
</template>
```

For more complex requirements, props controlled every aspect of behaviour — theming, validation rules, upload behaviour, sorting, editing, deletion and event handling. The component exposed `@select`, `@beforedelete`, `@delete`, `@upload` and other events for integrating with application-specific logic.

Upload-server examples for both PHP and Node.js were provided alongside the component, so developers had a working end-to-end reference without searching for separate documentation.

## Community and maintenance

The repository received 48 issues and 32 pull requests over its active life. Community contributions included bug fixes, feature improvements and translations. The project was used by developers worldwide and referenced in Vue ecosystem discussions as a recommended file upload solution.

Maintaining an open-source component at this level taught me that API design is the most important skill — every prop, event and slot must be justified because someone else will depend on its stability. Writing clear documentation and migration guides between releases was as important as writing the code itself.

## What I learned

Building Vue File Agent taught me more about component API design than any client project. Every prop, event and slot had to be justified — not just for current use cases, but for future ones I could not yet predict.

The project also demonstrated the value of developer tooling. Providing upload-server examples, comprehensive documentation and a CodePen playground significantly reduced the barrier to adoption. A component is only as good as the experience of integrating it.
